# Что сделано в задании 3

## Общая идея

В третьем задании я доработал галерею из задания 2 и добавил экспорт фото в общую системную галерею.

Теперь логика такая:

- исходный снимок по-прежнему хранится в приватной папке приложения;
- пользователь может открыть меню у карточки фото и выбрать `Экспорт в галерею`;
- на экране просмотра тоже появилась кнопка `Экспорт в галерею`;
- при экспорте создаётся копия файла уже в общей медиатеке устройства.

После успешного экспорта показывается `Snackbar`:

- `Фото добавлено в галерею`

## Что осталось из задания 2

Основа галереи не изменилась:

- фото снимаются через `TakePicture`;
- сохраняются в `getExternalFilesDir(Environment.DIRECTORY_PICTURES)`;
- список фото загружается при старте и после добавления нового снимка;
- сетка строится через `LazyVerticalGrid`.

## Как устроен экран

В приложении два экрана:

1. сетка всех фото;
2. экран просмотра одного снимка.

На главном экране:

- `LazyVerticalGrid` на 3 колонки;
- FAB-кнопка камеры;
- кнопка меню `⋮` на каждой карточке;
- пункт `Экспорт в галерею` в `DropdownMenu`;
- пустое состояние, если фото ещё нет.

На экране просмотра:

- само фото крупно;
- имя файла;
- дата;
- полный путь к файлу.
- кнопка `Экспорт в галерею`.

## Как хранятся фото

Имя файла формируется по условию:

- `IMG_yyyyMMdd_HHmmss.jpg`

Например:

- `IMG_20260412_183055.jpg`

Новый файл создаётся заранее в папке `Pictures` внутри app-specific storage, а потом уже его `Uri` передаётся в `TakePicture`.

## Как теперь работает экспорт

Экспорт — это не перенос исходного файла, а именно создание копии.

Это важно, потому что:

- оригинал должен остаться в приватной папке приложения;
- в общую галерею нужно положить отдельную копию.

## Почему используется MediaStore

Для современных Android правильный способ положить изображение в общую галерею — это `MediaStore`.

Я сделал так:

1. создаётся новая запись в `MediaStore.Images`;
2. через `OutputStream` туда копируются байты исходного `.jpg`;
3. после завершения запись становится видимой для системной галереи.

На старых Android до 29 я добавил совместимость:

- файл копируется в публичную папку `Pictures/Module5`;
- потом вызывается `MediaScannerConnection.scanFile(...)`, чтобы галерея увидела новый снимок.

## Как работает съёмка

Для запуска камеры используется:

- `ActivityResultContracts.TakePicture()`

Но камера не может просто так писать в приватный файл приложения по обычному пути. Поэтому я добавил:

- `FileProvider`

Он превращает файл в безопасный `content://` URI, который можно временно отдать внешнему приложению камеры.

## Почему нужен FileProvider

Если передавать путь как обычный `file://`, это считается устаревшим и небезопасным.

Поэтому схема такая:

1. приложение создаёт пустой файл;
2. через `FileProvider` получает URI;
3. этот URI передаётся в камеру;
4. камера записывает фото в этот файл.

## Как загружается список фото

В `GalleryViewModel` есть функция `reloadPhotos()`.

Она:

1. открывает папку `getExternalFilesDir(Environment.DIRECTORY_PICTURES)`;
2. берёт все `.jpg`-файлы;
3. сортирует их по дате изменения;
4. превращает каждый файл в `PhotoItem`.

По условию сканирование делается:

- при запуске приложения;
- после добавления нового фото.

Именно так здесь и реализовано.

## Главные классы и функции

### `GalleryViewModel`

Это главный класс логики экрана.

Он:

- хранит состояние галереи;
- загружает список фото;
- умеет найти конкретный снимок по имени файла.

### `reloadPhotos()`

Полностью перечитывает папку с фотографиями и обновляет состояние UI.

Для задания 2 это нормальный вариант, потому что в требованиях прямо сказано:

сканирование при запуске и при добавлении нового фото.

### `findPhoto(fileName)`

Находит конкретное фото для экрана просмотра.

### `GalleryApp(...)`

Корневой Compose-экран.

Здесь настроена навигация между:

- сеткой;
- просмотром фото.

### `GalleryScreen(...)`

Это главный экран сетки.

Внутри него:

- хранится временный `pendingPhotoUri`;
- хранится `pendingExportPhoto`, если ожидается экспорт после разрешения;
- объявлен launcher для камеры;
- объявлен launcher для разрешений;
- объявлен launcher для разрешения на экспорт на старых Android;
- есть функция `launchCamera()`.

### `launchCamera()`

Проверяет, какие разрешения ещё не выданы.

Если всё есть:

1. создаётся новый URI под будущее фото;
2. запускается камера.

Если разрешений не хватает:

сначала открывается системный запрос разрешений.

### `exportFromGrid(photo)`

Эта функция вызывается из пункта меню на карточке фото.

Что она делает:

- если на старом Android не хватает `WRITE_EXTERNAL_STORAGE`, сначала просит разрешение;
- если разрешение уже есть, сразу запускает экспорт;
- после успеха показывает `Snackbar`.

### `cameraLauncher`

Это обработчик результата камеры.

Если съёмка успешна:

- вызывается `onPhotoCaptured()`;
- `ViewModel` перечитывает папку;
- новое фото появляется в сетке.

Если съёмка отменена:

- временный файл удаляется;
- показывается `Snackbar`.

### `exportPermissionLauncher`

Нужен только для старых Android, где для записи в публичную папку ещё требуется разрешение на память.

После выдачи разрешения запускает отложенный экспорт именно того фото, которое пользователь только что хотел выгрузить.

### `createPhotoUri(context)`

Создаёт новый файл с правильным именем и возвращает `Uri` через `FileProvider`.

### `PhotoGridItem(...)`

Рисует одну ячейку сетки.

Для показа картинки используется `AsyncImage` из Coil.

Теперь у карточки есть меню:

- по кнопке `⋮` открывается `DropdownMenu`;
- внутри пункт `Экспорт в галерею`.

### `PhotoViewerScreen(...)`

Открывает одно выбранное фото на отдельном экране и теперь даёт экспорт прямо оттуда.

На этом экране экспорт работает через ту же общую функцию, что и на сетке.

### `EmptyPhotoState(...)`

Пустой экран, если фотографий ещё нет.

### `exportPhoto(...)`

Это удобная обёртка верхнего уровня.

Она:

1. уходит в `Dispatchers.IO`;
2. вызывает реальный экспорт;
3. по результату показывает `Snackbar` об успехе или ошибке.

То есть UI не работает напрямую с файловыми потоками.

### `exportPhotoToGallery(context, photo)`

Это главная функция экспорта.

Именно она делает копию файла в общую галерею.

Для Android 10+:

- использует `MediaStore`;
- создаёт новую запись;
- записывает байты фото;
- завершает сохранение.

Для Android 9 и ниже:

- создаёт файл в публичной папке `Pictures/Module5`;
- копирует туда байты;
- запускает медиасканер.

### `buildExportName(originalName)`

Делает имя экспортируемой копии.

Я добавил к имени суффикс `_export_...`, чтобы копия явно отличалась от исходного приватного файла и не конфликтовала с уже существующими файлами.

## Разрешения

Я добавил:

- `CAMERA`
- `WRITE_EXTERNAL_STORAGE` только для старых Android до 29

На новых версиях Android запись в app-specific папку не требует старого полного доступа к памяти.

## Что можно говорить на защите

Если спросят, почему исходное фото остаётся в приватной папке, можно ответить так:

Потому что экспорт в галерею должен создавать копию, а не переносить оригинал из app-specific storage.

Если спросят, зачем здесь `MediaStore`, можно ответить так:

Потому что это современный системный способ добавить изображение в общую галерею устройства.

Если спросят, зачем нужен `DropdownMenu`, можно ответить так:

По условию у каждого изображения в сетке должен быть пункт `Экспорт в галерею`, поэтому на карточке сделано меню действий.

Если спросят, почему экспорт сделан и в карточке, и на экране просмотра, можно ответить так:

Потому что по условию должна быть кнопка на экране просмотра, а меню у карточки — это дополнительная быстрая точка входа из сетки.
